■ Differential privacy

▼

Differential privacy provides a mathematical framework for quantifying privacy risk. By injecting calibrated statistical noise into queries or model outputs, it guarantees that any single individual's presence or absence in a dataset has a bounded effect on the result.

Key parameter: ε (epsilon) — the privacy budget. Smaller ε = stronger privacy but lower utility. Typical values range from 0.1 (strong) to 10 (weak).

Epsilon (ε) — Privacy BudgetThe core parameter of differential privacy. It bounds how much the output of a computation can differ based on one individual's data. A smaller ε (e.g. 0.1) gives a stronger privacy guarantee but requires adding more noise, reducing accuracy. A larger ε (e.g. 10) is weaker but more accurate. Setting ε is a deliberate privacy-utility trade-off.
Rényi Differential Privacy (RDP)An extension of standard DP that measures privacy loss using Rényi divergence — a generalization of KL divergence parameterized by order α. RDP offers tighter composition bounds when running multiple queries, making it especially useful for training machine learning models across many iterations (e.g. SGD with DP). Developed by Mironov (2017).
Local Differential Privacy (LDP)A variant of DP where each individual adds noise to their own data before it ever leaves their device or reaches any server. No trusted central curator is required. Apple and Google use LDP for collecting usage statistics. The trade-off: LDP typically requires more noise than global DP for the same accuracy, because there is no aggregation to average out individual noise.
Global (Central) Differential PrivacyThe classic DP model where a trusted curator collects raw data from individuals and adds noise to the aggregate query output before releasing it. Offers higher accuracy than Local DP for the same ε, but requires users to trust the data collector. Used in the U.S. Census Bureau's 2020 deployment.
Noise CalibrationThe process of choosing how much noise to add to a query output to satisfy a given ε guarantee. The amount of noise is proportional to the query's sensitivity — how much a single individual's data can change the result. Common noise distributions used are Laplace (for numeric queries) and Gaussian (for vector-valued outputs).
Composition TheoremsRules that describe how privacy loss accumulates when multiple DP mechanisms are applied to the same dataset. Basic composition: running k mechanisms each with ε gives total ε·k privacy loss. Advanced composition (and RDP) gives tighter bounds. Essential for understanding the true privacy cost of machine learning training with many iterations.

■ Cryptographic methods

▼

Cryptographic approaches enable computation on sensitive data without exposing the underlying values — critical in multi-party settings where no single party should see the combined dataset.

Secure Multiparty Computation (SMPC)A cryptographic protocol allowing multiple parties to jointly compute a function over their combined inputs without any party revealing their private input to others. Classic examples: Yao's Garbled Circuits (two-party) and the GMW protocol (multi-party). Used in privacy-preserving auctions, joint analytics across institutions, and federated statistics.
Homomorphic Encryption (HE)An encryption scheme that allows computations to be performed directly on ciphertext. The decrypted result matches what you would get from performing the same computation on the plaintext. Fully Homomorphic Encryption (FHE) supports arbitrary computations but is computationally expensive. Partially homomorphic schemes (like Paillier) support only addition or multiplication, but are much faster.
Zero-Knowledge Proofs (ZKP)A method by which one party (the prover) can prove to another (the verifier) that a statement is true without revealing any information beyond the truth of the statement itself. Example: proving you know a password without revealing it. Used in blockchain privacy (e.g. zk-SNARKs), identity verification, and regulatory compliance without data disclosure.
End-to-End Encryption (E2EE)A communication system where only the communicating users can read the messages. The service provider transmits encrypted data but cannot decrypt it. Used in messaging apps (Signal, WhatsApp), secure email, and file storage. E2EE protects data in transit and at rest from third parties, including the platform itself.

■ Federated learning

▼

Federated learning trains ML models across decentralized devices holding local data — only model gradients are shared, never raw data. Especially valuable in healthcare and mobile settings.

Key constraints: high communication costs, non-IID data distributions, and gradient inversion attacks. Often paired with differential privacy for stronger guarantees.

Cross-Device Federated LearningFL deployed across a very large number of edge devices (e.g. smartphones). Each device trains locally on its own data and sends only model updates. Involves millions of clients, high dropout rates, and heterogeneous hardware. Google's Gboard and Apple's on-device models use this approach.
Cross-Silo Federated LearningFL across a small number of organizations or data silos (e.g. hospitals, banks). Each silo holds a large dataset and participates reliably. Used in healthcare consortia to train models on patient data across multiple institutions without sharing records. Easier to coordinate than cross-device FL but requires data governance agreements.
Gradient AggregationThe central step in federated learning: each participant computes gradients on their local data and sends them to a server, which aggregates them (typically via averaging) to update the global model. Secure aggregation protocols ensure the server only sees the aggregate, not individual gradients, providing additional privacy guarantees.
Gradient Inversion AttacksA class of attacks (e.g. Deep Leakage from Gradients, Geiping et al.) that can reconstruct training data from shared gradients. An adversarial server (or man-in-the-middle) can recover near-pixel-perfect images or sensitive text from gradient updates alone. This is why FL alone is not sufficient for privacy — pairing with DP or secure aggregation is recommended.

■ Privacy laws & legal considerations

▼

Privacy is a legal challenge as much as a technical one. Organizations must navigate regulations that vary by jurisdiction and sector.

General Data Protection Regulation (GDPR) — EU, 2018The world's most comprehensive data protection law, applying to any organization handling EU residents' data. Key rights: access, erasure, portability, objection to automated decisions. Requires a lawful basis for processing, Data Protection Impact Assessments (DPIAs) for high-risk activities, and data breach notification within 72 hours. Penalties: up to €20M or 4% of global annual turnover.
California Consumer Privacy Act (CCPA) — California, 2020Gives California residents the right to know what personal data is collected, the right to delete it, and the right to opt out of its sale. The CPRA (2023 amendment) added a right to correct inaccurate data, restricted use of sensitive personal information, and created the California Privacy Protection Agency. Applies to businesses meeting certain size/revenue thresholds.
Health Insurance Portability and Accountability Act (HIPAA) — US, 1996Governs the use and disclosure of Protected Health Information (PHI) by covered entities (healthcare providers, insurers) and their business associates. The Privacy Rule defines what constitutes PHI and when it may be shared. The Security Rule mandates technical safeguards for electronic PHI. Violations can result in civil penalties up to $1.9M per violation category per year.
Children's Online Privacy Protection Act (COPPA) — US, 1998Restricts the collection of personal information from children under 13 by online services. Requires verifiable parental consent before collecting, using, or disclosing a child's data. Operators must maintain a clear privacy policy and allow parents to review or delete their child's data. Enforced by the FTC; violations can result in civil penalties up to $51,744 per violation.
Personal Information Protection and Electronic Documents Act (PIPEDA) — Canada, 2000Canada's federal private-sector privacy law. Requires organizations to obtain meaningful consent for data collection, use, and disclosure; limit collection to what's necessary; and protect personal information with appropriate safeguards. Being updated by Bill C-27 (Consumer Privacy Protection Act), which would introduce stronger consent requirements and significant penalties.
Personal Data Protection Act (PDPA) — Singapore, 2012Singapore's main data protection law, covering the collection, use, and disclosure of personal data by private organizations. Amended in 2020 to introduce mandatory breach notification (within 3 days for significant breaches), increased financial penalties (up to S$1M or 10% of annual Singapore turnover), and new provisions for data portability and deemed consent.
Data MinimizationA core privacy principle (enshrined in GDPR Article 5) requiring that only the minimum necessary personal data be collected for a specified, explicit, and legitimate purpose. Organizations should not collect data "just in case" it might be useful later. Technically, this aligns with differential privacy's idea of limiting exposure — collect less, risk less.
Right to Erasure ("Right to be Forgotten")Under GDPR Article 17, individuals can request deletion of their personal data when it is no longer necessary for the original purpose, consent is withdrawn, or the data was unlawfully processed. The landmark case Google Spain v. AEPD (2014) established this right in EU law. Technical challenge: deletion from ML models trained on personal data is an active research area ("machine unlearning").
Non-Compliance PenaltiesGDPR: up to €20M or 4% of global annual turnover (whichever is higher). Notable fines: Meta €1.2B (2023), Amazon €746M (2021), WhatsApp €225M (2021). CCPA: $2,500 per unintentional violation, $7,500 per intentional violation. HIPAA: $100–$50,000 per violation, up to $1.9M annually. Criminal liability possible for intentional violations.

Notable court cases: Google LLC v. CNIL (2019) — CJEU ruled right to erasure applies within EU only, not globally. Meta GDPR fine (2023) — €1.2B for unlawful transfer of EU user data to the US. Illinois BIPA litigation — multiple class actions (Facebook $650M, TikTok $92M) for biometric data collection without consent.

Why anonymisation fails
The limits of de-identification
Supposedly anonymous datasets can be re-identified using auxiliary information. As few as 3–4 quasi-identifiers can uniquely identify most individuals in a dataset.

Anonymisation is not a privacy guarantee — it is a risk reduction heuristic. Without mathematical privacy guarantees (like DP), re-identification is almost always possible given enough auxiliary data.

Re-identification attacks

Combining anonymized data with public datasets can uniquely identify individuals.

Quasi-identifiers

Non-identifying attributes in combination (ZIP + age + gender) can pinpoint a person.

ML inference attacks

Models trained on anonymized data can be probed to reveal membership or sensitive attributes.

Case studies
Location data

NYC Taxi data (2014)

A dataset released under FOIA with hashed medallion numbers was reverse-engineered by researchers, who then cross-referenced celebrity photos to trace individual taxi trips and infer driver earnings.

Re-identification

Netflix Prize dataset (2009)

Narayanan & Shmatikoff showed that 8 movie ratings plus approximate dates were sufficient to uniquely identify most users in Netflix's anonymized dataset, by cross-referencing IMDb reviews.

Behavioral data

AOL search query release (2006)

AOL released 20 million queries with user IDs replaced by numbers. New York Times journalists identified specific individuals within days — behavioral patterns alone are identifying, even without names.

Health data

Massachusetts medical records (Sweeney, 1997)

Latanya Sweeney re-identified the Massachusetts governor's medical records from a "de-identified" state employee health database using only ZIP code, birth date, and sex — all available in the voter registration list.

Success stories
When privacy works
Privacy-preserving technologies are already powering real-world systems at scale.
Federated learning

Google Gboard (2016–present)

Google uses federated learning to improve next-word prediction on Gboard without sending keystrokes to central servers. Model updates are computed on-device, aggregated securely, and released with differential privacy — a landmark deployment at consumer scale.

Differential privacy

Apple's usage statistics (2016–present)

Apple applies local differential privacy to collect emoji usage, QuickType suggestions, and crash data from hundreds of millions of devices. Each device adds noise before transmission, so no individual's behavior can be reconstructed.

Differential privacy

U.S. Census Bureau (2020)

The 2020 U.S. Census deployed differential privacy at unprecedented scale, using the TopDown algorithm to release demographic statistics while bounding individual disclosure risk — the largest real-world DP deployment to date.

SMPC + Cryptography

COVID-19 contact tracing — DP-3T (2020)

The Decentralized Privacy-Preserving Proximity Tracing protocol kept all contact data on-device and used cryptographic identifiers. Users were notified of exposure without revealing identities of infected contacts to any central server.
